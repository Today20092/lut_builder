# Domain docs

Before exploring the codebase, read these when they exist:

- `CONTEXT.md` at the repository root
- `docs/adr/` for relevant architectural decisions

Missing files are expected; proceed silently. Domain-modeling skills create them only when needed.

## Layout

This is a single-context repository:

- Domain terminology belongs in `/CONTEXT.md`.
- Architectural decisions belong in `/docs/adr/`.

Use terminology defined in `CONTEXT.md`. If proposed work conflicts with an ADR, identify that conflict explicitly.
